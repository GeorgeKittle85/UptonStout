How Facial Recognition Works — Technical Guide

      
Classical computer vision vs modern deep learning, explained

    

    

  


  
Facial recognition pipelines combine image processing, geometry, and machine learning. This guide walks through every stage — from detecting a face in raw pixels to making a final match/no-match decision — contrasting historical classical methods with modern neural approaches.

Face Detection — Finding the Face in an Image

    Classical: Viola-Jones
    Neural: MTCNN / SSD
    ▼
  

  

    
What is Face Detection?

    
Before recognition can happen, the system must locate all faces in the image. This is a binary classification problem (face vs background) repeated at every position and scale in the image using a sliding window approach.

    
🕰 Classical: Viola-Jones (2001)

    
Three core innovations made it the first real-time face detector:

    
    
      
  • Haar-like features: Simple rectangular brightness-difference patterns. Eyes are darker than cheeks; the nose bridge is lighter than sides. 160,000+ such features exist per detection window.
    • 
      
  • Integral Image: A precomputed lookup table so any rectangular region sum costs exactly 4 operations — O(1). Without it, evaluating all Haar features would be impossibly slow.
    • 
      
  • AdaBoost + Cascade: AdaBoost picks the 200 most discriminative features. They're arranged in a cascade — cheap early stages reject 99.9% of non-face regions before expensive later stages run.
    • 
    

    
⚡ Viola-Jones ran at 15 fps on 2001 hardware. Still shipped in OpenCV today. Its main limitation: only reliable for near-frontal faces within ±30° tilt.

    
🤖 Neural: MTCNN / SSD MobileNet

    
Modern CNNs learn features hierarchically from millions of training images — edges → facial parts → faces — entirely from data. They handle rotation to ±45°, occlusion, diverse lighting, and varied ethnicity with 97–99% accuracy vs Viola-Jones' ~88%.

Facial Landmarks — Mapping 68 Key Points

    Classical: ASM / AAM
    Neural: CNN Regression
    ▼
  

  

    
What are Landmarks?

    
Landmarks are precise (x,y) coordinates at anatomical locations: 17 jaw points, 10 eyebrow points, 9 nose points, 12 eye points, 20 mouth points. They enable two things:

    
    
      
  • Face alignment: Rotate and scale so eye centers always land at fixed positions, removing the largest sources of pose variation before recognition
    • 
      
  • Geometric analysis: Compute classical ratios — face aspect, interpupillary distance, nose-mouth proportions
    • 
    

    
🕰 Classical: Active Shape Models (Cootes, 1995)

    
ASMs learn a statistical shape model of valid face configurations via PCA. Fitting iteratively moves landmarks toward strong image gradients while staying within ±3 standard deviations of the learned shape distribution — preventing physically impossible configurations.

    
🤖 Neural: CNN Regression (Hourglass Networks)

    
A CNN trained end-to-end directly outputs all 68 (x,y) values from a cropped face crop. Stacked Hourglass networks progressively refine predictions — similar in spirit to ASM's iterative fitting but learned entirely from data. Modern accuracy: <1px mean error on benchmarks.

    
📐 Affine alignment: Compute the angle θ between eye center coordinates, rotate the image by −θ, then scale so the inter-eye distance equals a fixed target (e.g., 72px). This normalization step is why recognition works across different photo distances.

Feature Extraction — Encoding Identity as Numbers

    Classical: Eigenfaces / HOG / LBP
    Neural: ResNet Embeddings
    ▼
  

  

    
The Core Challenge

    
Convert a face image into a compact vector where same-person faces cluster together and different-person faces are far apart — across all lighting, expression, and pose variations.

    
🕰 Eigenfaces — PCA (Turk & Pentland, 1991)

    
Apply PCA to a database of flattened face images. Each image is a point in high-dimensional space (e.g. 150×150 = 22,500 dims). PCA finds the "eigenfaces" — directions of maximum variance — and projects each new face onto the top K eigenvectors:

    
f = Wᵀ(x − μ)   where W = [e₁, e₂, ..., eₖ]

    
Works well in controlled studio lighting. Fails with real-world illumination variation because PCA captures lighting changes as identity differences.

    
🕰 HOG — Histogram of Oriented Gradients

    
Divide image into 8×8 pixel cells. For each pixel, compute gradient direction and magnitude. Build a 9-bin histogram of directions per cell. Normalize across 2×2 cell blocks for illumination invariance. Concatenate all histograms into one feature vector. A linear SVM on HOG achieves ~97% face detection accuracy.

    
🤖 Neural: ResNet Embeddings with Triplet Loss (FaceNet, dlib)

    
A deep residual network trained using triplet loss on millions of face pairs. Given anchor A, positive P (same person), negative N (different person):

    
L = max(0, d(A,P) − d(A,N) + α)

    
The loss forces same-person pairs closer and different-person pairs further apart by margin α. After training, the network outputs a 128-d embedding where Euclidean distance directly measures identity similarity.

    
✦ This tool uses Claude's multimodal vision model as the "feature extractor" — its internal representations, built from pretraining on vast image datasets, act as a very large embedding network that can also articulate its reasoning in natural language.

Matching & Decision — Making the Call

    Distance Metrics + Thresholds
    ▼
  

  

    
Distance Metrics

    
Euclidean Distance (standard for dlib ResNet-34):

    
d(a,b) = √( Σᵢ (aᵢ − bᵢ)² )

    
Cosine Similarity (scale-invariant, good for normalized embeddings):

    
cos(a,b) = (a · b) / (‖a‖ · ‖b‖)

    
Standard Thresholds (128-d Euclidean)

    

      
DistanceInterpretation

      
0.0 – 0.3Very high confidence match

      
0.3 – 0.5Confident match

      
0.5 – 0.6Probable match (boundary zone)

      
0.6 – 0.8Likely different person

      
0.8+Confident different person

    

    
Why Average Multiple Reference Photos?

    
Single photos carry per-image noise (lighting, expression, angle). Averaging N descriptors produces a centroid more representative of true identity:

    
d = ‖ (1/N)·Σ refᵢ − comp ‖

    
This is nearest-centroid classification — one of the simplest yet most effective classifiers in ML, dating back to the 1960s.

    
Limitations

    
    
      
  • Demographic bias: Models trained on imbalanced data show higher false-positive rates for underrepresented groups
    • 
      
  • Identical twins: Genetic similarity pushes descriptors below threshold — they will match each other
    • 
      
  • Aging: 10+ year gaps reduce accuracy; separate age-invariant models exist (AgeDB benchmark)
    • 
      
  • Quality: Face width <80px, motion blur, or >45° pose angle all significantly degrade accuracy
    • 
    

    
⚖ Ethics: Facial recognition must only be used with full consent of all subjects. This tool processes images locally via API — no photo data is retained after the API call returns.

What Makes Claude's Analysis Different from Classic Methods

    
Unlike a single Euclidean distance score, Claude evaluates multiple independent cues and combines them — analogous to how ensemble learning (boosting, bagging) combines weak classifiers into a stronger one:

    
    
      
  • Structural geometry: jaw, cheekbones, cranial width, facial thirds
    • 
      
  • Specific features: eye shape/spacing, nose bridge, mouth morphology, ears
    • 
      
  • Contextual factors: lighting variation, expression change, grooming, aging
    • 
      
  • Holistic gestalt: overall impression weighted by reliability of each cue
    • 
    

    
The natural language output also provides interpretability — a property that single-metric distance scores fundamentally lack. This is the key advantage of neuro-symbolic and explainable AI approaches over pure black-box embedding systems.
